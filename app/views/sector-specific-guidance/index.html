How to format EA data returns

From:
Environment Agency
First published:
September 2016

Follow rules to format data that meets Environment Agency standards then send your data returns online.

Contents

  2. Sector specific data formatting rules

General data formatting rules

Before you send your EA data returns file you need to apply data formatting rules under fixed field headings. This will help you meet Environment Agency data standards, reduce ambiguity and improve consistency in your data returns.

You can set up your data management systems to produce an automated export of data in the correct format then you need to save your file as CSV to send it online.

You can use the online service to submit data covering up to 18 months in the past.

Sector specific data formatting rules

These rules explain how you must prepare and present compliance monitoring data according to each field heading in your file.

You must use the short field headings in your file. You can send data under the short headings in any order.

Some fields are mandatory for all data returns. Where possible you should give data for other fields as specified in your permit or licence and particular type of return.

Some fields only accept entries from the Environment Agency controlled lists of standard terms, which you can download to apply in your software.
