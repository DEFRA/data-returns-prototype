Version 1.1

Version 1.1 was our first version prototype for all-sectors.

A hypothesis based on user needs gathered from interviews with users who submit PI data and business requirements.

Summary of Version 1.1

Reason/source

Version 1.1 was a copy of version 1.0 with the data updated for a Paper and Pulp manufacturer.

We hypothesised we could make the service simple by presenting the fewest data input points for most users.

The data entry user interface is a summary page with the previous year’s submission data preloaded (substance and value) and the current threshold. Input options allow for value and unit only. In the event the values to be inputted are exceptional, users can follow links to add more detail.

Any QA issues are highlighted in the UI when the user continues. Following “more detail” will reveal more messaging about the issue and allow users to correct or explain the data. Common issues are shown plus an ‘other’ where users can add their own comments.

“Keep it simple”

Users tend to be specialists and are likely to transpose data that they have stored in their own spreadsheet.

Users need to see historical data. They often input the same data each year and they tend to forget what they submitted the previous year.

Users have a preference for handling data manually.

Version 1

Version 1 was our first version prototype using the gov.uk prototype kit. It is a high-fidelity mock-up with which users can interact. It is based on a user journey built in sketch

An initial hypothesis based on user needs gathered from interviews with users who submit PI data and business requirements.

Summary of Version 1

Reason/source

Version 1 demonstrated the user journey for an intensive farmer.

We hypothesised it would be simpler for an intensive farmer if, rather than asking the user to determine their site's emissions using the formula given in the guidance, users could count livestock and how they are housed and the service could do the calculations to determine site emissions.

This differs from all other sectors which requests the substances and amounts to be measured, calculated estimated by the user.

Until we can draw business requirements from the internal needs research we assumed all the data being requested on the Pollution Inventory reporting form is required so company details, sharing information is collected.

We did not include user authentication

The intensive farming sector has specific user needs that differ from all other sectors and has the most sites and users of all sectors - they were also the first user the prototype was tested on.

High level user needs included in this version:
